## 수집 흐름

```
League API → 시드 플레이어 수집 (Challenger/GM/Master만 직접 조회)
    ↓
Match History API → 매치 ID 폴링 (5분 주기)
    ↓
Match Detail API → 매치 상세 데이터 수집 (1건당 8명 데이터)
    ↓
매치 참가자의 puuid → Snowball 확장 (Platinum 이상만 시드 추가)
    ↓
DB 적재 → 티어별 통계 집계
```

> Platinum/Diamond은 League API로 직접 수집하지 않음 — Snowball로 자연 확보 (API 비용 절약 + 활동 유저만 확보)

## API Key 단계별 전략

| 단계 | 키 | 제한 | 수집 규모 |
| --- | --- | --- | --- |
| 개발/초기 수집 | Development Key | 100 req/2분, 24시간 만료 | 테스트용, 지속 수집 불가 |
| 프로토타입 완성 후 | Personal Key | 100 req/2분, 만료 없음 | 하루 ~30,000 매치 수집 가능 |
| 심사 통과 후 | Production Key | 30,000 req/10분 | 본격 서비스 운영 |

> Personal Key 기준으로 파이프라인 설계, Production Key 전환 시 스케일업

## Personal Key 기준 수집 계획

- **분당 처리량**: 50 req/분 (100 req/2분)
- **매치 1건 수집 비용**: 매치 상세 1 req (8명 데이터 포함)
- **매치 ID 조회 비용**: 시드 플레이어 1명당 1 req
- **실질 수집량**: 매치 ID 조회에 30%, 매치 상세에 70% 배분 → 분당 약 35 매치 상세
- **일일 수집량**: 이론치 72,000 req/하루, 에러 재시도/대기 감안 실질 ~60,000 req → 약 35,000~40,000 매치

## 수집 대상

- **통계 기준 티어**: Platinum 이상
- **시드 풀**: Challenger/GM/Master로 시작 → Snowball로 Platinum까지 확장
- **Snowball 확장 규칙**: 수집한 매치의 참가자 중 Platinum 이상만 시드 풀에 추가
- **리전**: KR 우선, 이후 글로벌 확장
- **패치 구분**: `game_version` 필드에서 패치 버전 추출, 패치별 데이터 분리 저장

## 폴링 전략

- **폴링 주기**: 시드 플레이어별 5분 간격으로 새 매치 확인
- **중복 방지**: DB에 `matchId` 유니크 제약, 이미 수집한 매치 ID는 스킵
- **우선순위**: 고티어 시드를 더 자주 폴링 (Challenger/GM: 3분, Master: 5분, 이하: 10분)
- **비활성 시드 정리**: 7일간 새 매치 없는 시드는 폴링 대상에서 제외

## 패치 전환 처리

- **패치 감지**: Data Dragon 버전 API를 1시간 주기로 확인
- **전환 시점**: 새 패치 감지 시 이전 패치 데이터 수집 계속 (진행 중인 매치 마무리)
- **통계 분리**: 새 패치 데이터는 별도 집계, 기존 패치 통계는 아카이브
- **최소 데이터 기준**: 새 패치 데이터 1,000 매치 미만이면 "데이터 수집 중" 표시

## 초기 부트스트랩

런칭 전 최소 데이터 확보 전략:

- **목표**: 최소 10,000 매치 (통계 유의미한 최소 기준)
- **소요 시간**: Personal Key 기준 약 8~12시간
- **방법**: 시드 풀(~4,000명)의 최근 20매치 ID를 일괄 수집 → 유니크 매치 상세 수집
- **완료 기준**: 주요 덱 클러스터가 10개 이상, 각 클러스터 최소 100 매치 이상

## 에러 처리

| 상황 | 처리 |
| --- | --- |
| 429 (Rate Limit) | `Retry-After` 헤더 기반 대기 후 재시도 |
| 404 (매치/소환사 없음) | 스킵, 해당 ID를 실패 목록에 기록 |
| 5xx (서버 에러) | 지수 백오프로 최대 3회 재시도 |
| API 장애 | Status API로 확인, 장애 중 수집 일시 중단 |
| 키 만료 (Dev Key) | 알림 발송, 수집 자동 중단 |

> API 상세 스펙은 [[Riot TFT API]] 참고
