## 개요

Riot TFT API는 게임 종료 시점의 최종 보드 스냅샷만 제공하며, 라운드별 진행 데이터는 없음. AI 피드백 기능은 이 제약 안에서 설계해야 한다.

## 활용 가능한 데이터

매치 종료 후 참가자별로 수집 가능한 필드:

| 데이터 | 필드 | 비고 |
| --- | --- | --- |
| 최종 등수 | `placement` | 1~8위 |
| 최종 레벨 | `level` | |
| 남은 골드 | `gold_left` | 최종값만 |
| 마지막 라운드 | `last_round` | |
| 탈락 시점 | `time_eliminated` | 초 단위 |
| 증강 | `augments[]` | 3개, 배열 순서 = 선택 순서 |
| 챔피언 | `units[]` | ID, 성급, 아이템(최대 3개) |
| 특성 | `traits[]` | 활성 여부, 티어 포함 |
| 총 피해량 | `total_damage_to_players` | |
| 처치 수 | `players_eliminated` | |
| 킬/골드/딜량 | `missions` (Kills, GoldEarned, DamageDealt) | 추가 통계 |

> 전체 API 스펙은 [[Riot TFT API]] 참고

## 가능한 피드백

### Tier 1 — 통계 비교로 구현 가능

**덱 메타 평가**
- 선택한 덱의 현 패치 승률/평균 등수를 알려줌
- 비슷한 시너지의 더 나은 덱 제안
- 예시: "이 덱은 현 패치 평균 4.8등. 비슷한 시너지의 X 덱(평균 3.2등)이 더 나았을 수 있어요"

**아이템 빌드 최적화**
- 같은 덱 상위권 플레이어의 아이템 조합과 비교
- 예시: "같은 덱 1~2등 유저들은 이 챔피언에 A/B/C를 줬는데, 당신은 D를 줬어요"

**증강 선택 리뷰**
- 덱별 증강 승률 기반 평가, 더 나은 선택지 제시
- 예시: "2차 증강 X를 골랐는데, 이 덱에서는 Y 증강이 승률 12% 높습니다"

### Tier 2 — DB 쿼리 + 비교 로직 필요

**유사 매치 벤치마크**
- DB에서 비슷한 조합의 고등수 매치를 찾아 차이점 하이라이트
- 예시: "비슷한 구성으로 1등한 매치 3건 — 차이점: 아이템 X, 증강 Y"

**챔피언 구성 피드백**
- 핵심 유닛 포함 여부, 불필요 유닛 지적, 대체 유닛 제안
- 예시: "핵심 유닛 A가 빠져있어요. 상위권의 80%가 A를 포함합니다"

**성급/레벨 효율 분석**
- 최종 레벨, 성급 분포를 같은 덱 상위권과 비교
- 예시: "레벨 8에서 2성 위주인데, 같은 덱 1등들은 평균 레벨 9 + 핵심 3성 1개"

### Tier 3 — LLM 통합

**종합 매치 리포트**
- 위 항목을 종합하여 LLM 기반 자연어 피드백 생성
- 덱 선택, 아이템, 증강, 구성을 한 번에 분석한 종합 리포트

## 불가능한 피드백

| 분석 유형 | 이유 |
| --- | --- |
| 리롤/레벨업 타이밍 조언 | 라운드별 골드/경험치 데이터 없음 |
| 이코노미 운영 평가 | 라운드별 골드 추적 불가 |
| 보드 배치 피드백 | 유닛 위치 데이터 미제공 |
| 라운드별 의사결정 분석 | 라운드별 진행 데이터 없음 |
| 상점 선택 최적화 | 상점 데이터 미제공 |

## 부분적으로 추론 가능

| 분석 유형 | 추론 방법 | 정확도 |
| --- | --- | --- |
| 레벨링 속도 추정 | `last_round` + `level` + `time_eliminated` 조합 | 중 |
| 덱 방향 전환 여부 | traits 불일치도 (산발적 특성 = 전환 실패 가능성) | 중 |
| 캐리 전환 시점 추정 | 아이템 분포 패턴 (저코 캐리→고코 캐리) | 하 |
